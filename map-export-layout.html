Map Export Layout Template

⚠️ This is a live template! The export system loads this HTML file and uses its structure and styling for all map exports (PDF, PNG, JPEG, GeoTIFF). You can customize the export appearance by editing the CSS in this file.

How to Customize

Simply edit the CSS styles in the <style> section above. Changes will be reflected in all future exports. The template uses data attributes to populate content:

  • data-export-qr - QR code container (auto-sizes based on URL length, contains full unmodified URL)
  • data-export-title - Map title (single line with ellipsis)
  • data-export-attribution - Data attribution
  • data-export-info - Formatted export info: timestamp | base URL | params (selection layer completely filtered out including geojson, URL decoded, separated by &) | hash. Text wraps as needed to show full URL.
  • data-export-scale-label - Scale bar distance label (right end, e.g., "1 km"). Left end shows "0" by default.
  • data-export-north-arrow - North arrow (rotates with map bearing)
  • data-export-layer-thumbnails - Container for active layer thumbnails (dynamically populated using LayerThumbnail.generate() from js/layer-thumbnail.js)

Layer Thumbnails

The layer thumbnails are automatically generated from the active map layers using the LayerThumbnail class. Each row shows:

  • 10mm × 10mm visual preview of the layer style (circles, lines, polygons, or raster patterns)
  • Layer title text (6pt, up to 2 lines with ellipsis)
  • Layer type badge (Vector, GeoJSON, CSV, Raster, Style)
  • Out-of-view indicator if layer data is not visible at current map extent

Thumbnails are arranged vertically in rows with 1.5mm gap between rows, 2mm gap between thumbnail and title (max width: 50mm).

Implementation: The export control script (js/map-export-control.js) iterates through active layers, creates a row container, generates a thumbnail using LayerThumbnail.generate(layer, 38, options), adds the layer title, and appends to the container with data-export-layer-thumbnails attribute.

Layout Structure

This template shows the layout structure for all map export formats (PDF, PNG, JPEG, GeoTIFF). The layout creates a unified, compact metadata section at the bottom of the map:

  • Bottom Left Section: Text metadata (title, attribution, export info) stacked vertically in white box
  • Bottom Right Top: QR code in its own white box
  • Bottom Right Bottom: Legend box containing:
    • Legend header with "LEGEND" title and north arrow side by side
    • Scale bar with label and visual representation
    • Active layer thumbnails arranged in rows

Key Measurements:

  • Bottom margin: 4mm from edges
  • Gap between sections: 3mm
  • Gap within sections: 2-4mm
  • Box padding: 6-10px (reduced for compact layout)
  • Element backgrounds: white
  • Title font: 12pt bold, single line with ellipsis, dark gray (#1a1a1a)
  • Attribution font: 7pt, medium gray (#333)
  • Export info font: 6pt, light gray (#666)
  • Legend title font: 9pt bold uppercase, dark gray (#1a1a1a)
  • QR code size: 20mm × 20mm (auto-sized)
  • Scale bar: 30mm wide × 4mm tall with borders, center divider, and labels at both ends (0 and distance)
  • North arrow: 10mm × 10mm, simple upward-pointing triangle
  • Layer thumbnails: 10mm × 10mm each with layer title, arranged vertically with 1.5mm gap, max container width 50mm

Design Principles:

  • Three-section layout: text metadata (left), QR code (top right), legend (bottom right)
  • Clean white backgrounds for all elements
  • Single-line title with ellipsis prevents awkward line breaks
  • Strong typographic hierarchy (title → attribution → metadata → legend)
  • Professional black-on-white color scheme for universal readability
  • Legend box organizes cartographic elements (north arrow, scale bar) with active layers
  • Layer thumbnails provide visual reference for active map layers
  • Improved scale bar with clear visual representation (borders and center divider)
  • Compact north arrow integrated into legend header
  • QR code isolated for better scannability
